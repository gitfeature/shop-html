# shop-html 项目问题汇总

## 一、安全问题

### 1. XSS 漏洞
- **位置**: `js/auth.js`, `js/cart.js`
- **问题**: 用户输入内容未经过滤直接插入到 DOM 中
- **影响**: 可能导致跨站脚本攻击
- **建议修复**:
  - 对所有用户输入进行 HTML 转义
  - 使用 textContent 而不是 innerHTML

### 2. localStorage 数据泄露
- **位置**: `js/cart.js`
- **问题**: 购物车数据存储在 localStorage 中未加密
- **建议修复**:
  - 对敏感数据进行加密存储
  - 添加数据签名防止篡改

## 二、逻辑错误

### 1. 结算页面链接错误
- **位置**: `checkout.html` 第22行
- **问题**: 链接指向购物车页面而非结算确认页面
- **当前代码**: `<a href="cart.html" class="active">购物车</a>`
- **建议**: 修改为 `<a href="checkout.html" class="active">结算</a>`

### 2. 购物车数量更新逻辑
- **位置**: `js/cart.js` 第40-47行
- **问题**: 添加商品时，未检查数量是否超过库存
- **建议**: 添加最大购买数量限制

### 3. 选项比较不严谨
- **位置**: `js/cart.js` 第40-44行
- **问题**: 直接访问 `options.size` 和 `options.color`，当 options 为空或 undefined 时会报错
- **建议**: 使用可选链 `options?.size`

## 三、用户体验问题

### 1. 错误提示不友好
- **位置**: 多个 JS 文件
- **问题**: 错误直接输出到 console，未向用户展示友好提示
- **建议**: 使用 toast 组件或模态框向用户展示错误信息

### 2. 缺少加载状态
- **问题**: 数据加载和操作过程中没有加载中的视觉反馈
- **建议**: 添加 loading 动画或禁用操作按钮

### 3. 表单验证缺失
- **位置**: `login.html`, `checkout.html`
- **问题**: 表单提交前未进行客户端验证
- **建议**: 添加必填项检查、格式验证等

## 四、代码规范问题

### 1. 重复代码
- **位置**: 多个 HTML 文件
- **问题**: 头部导航和底部结构在每个页面重复定义
- **建议**: 提取为公共组件

### 2. 硬编码问题
- **位置**: 多个文件
- **问题**: 魔法数字、硬编码 URL 等
- **建议**: 使用配置文件或常量定义

## 五、性能问题

### 1. 资源未压缩
- **问题**: 图片资源未进行压缩优化
- **建议**: 使用 WebP 格式，压缩图片文件大小

### 2. 重复请求
- **问题**: 每次页面加载都从 localStorage 读取数据，未做缓存
- **建议**: 合理使用内存缓存

## 六、待优化功能

1. 搜索功能不支持中文分词
2. 商品列表不支持排序和筛选
3. 用户密码未加密传输（如果有登录功能）
4. 缺少深色模式支持
